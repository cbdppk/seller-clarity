# Git PR Coordination Skill

Use this skill when coordinating parallel hackathon work.

## Purpose
Keep three teammates moving without blocking each other.

## Guardrails
- One repo.
- Short-lived branches.
- No one works directly on main.
- Merge early, not only at the end.
- Do not change `/lib/contracts.ts` casually.

## Standards
- Branch names should describe the task.
- Small PRs are better than huge PRs.
- The coordinator merges after basic checks.
- Pull latest main before opening a new branch.

## Deliverables
- Clean branch workflow.
- Predictable ownership.
- Fewer integration conflicts.
