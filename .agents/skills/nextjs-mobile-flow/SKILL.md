# Next.js Mobile Flow Skill

Use this skill when building the product flow in the seller-clarity hackathon app.

## Purpose
Build a clean, mobile-first Next.js App Router flow for a one-day hackathon MVP.

## Guardrails
- Keep the app text-first first, image-second later.
- Prefer simple page flow: Home -> Analyze -> Results.
- Use card-based layout.
- Optimize for small screens first.
- Do not add auth, database, roles, or inventory.

## Standards
- Use TypeScript.
- Keep components small and predictable.
- Use shared types from `/lib/contracts.ts`.
- Use loading, empty, and error states.
- Keep copy short and clear for non-technical sellers.

## Deliverables
- Home page with textarea, helper text, and sample chips.
- Results page with summary cards, records, charts, and insights.
- Navigation that preserves the happy path.
