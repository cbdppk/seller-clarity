# Recharts Demo Polish Skill

Use this skill when building seller-facing result views and demo polish.

## Purpose
Make the analysis easy to understand on a phone screen.

## Guardrails
- Charts must be readable on mobile.
- Hide charts gracefully if the required data is missing.
- Keep labels short.
- Never clutter the screen.

## Standards
- Use summary cards first.
- Prefer one bar chart and one pie/donut chart max.
- Use clear empty/loading/error states.
- Add a sample-data fallback that always works.
- Write plain-language helper text.

## Deliverables
- Summary cards.
- Revenue bar chart.
- Optional channel pie chart.
- Microcopy.
- Demo-ready polish.
